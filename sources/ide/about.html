About the 6502 IDE

The 6502 IDE started off as an experiment to work with WASM and, more directly, the emscripten project.

It was directly inspired by http://6502asm.com/. My original goal was to replace the compiler with one more powerful as I wanted a more flexible playground to experiment with. I soon saw that it was very easy to replace the 6502 Emulator as well.

Meet the assembler: DASM

I choose dasm as my assembler of choice, mostly because I have worked with it on other projects and was quite comfortable with the tool.

The dasm source is completely original with the sole exception of adding a web_main function that allows me to call main. I hope, with a bit more experience, I'll be able to get rid of web_main.

I do use a custom makefile, mostly to set some emscripten specific things.

Meet the 6502: fake6502

I use the excellent fake6502, (c)2011 Mike Chambers (miker00lz@gmail.com) TThis is another package I have experience with, and it has a good reputation in the community. The main module took no modification to compile, but I did have to include a few helper functions.

The result

The result is the first release of 6502 IDE

While the compiler is slightly slower than the origional javascript compiler, it is still very quick. The compiler is far more powerful than any compiler I have found on a web-based 6502 emulator.

The origional JavaScript emulator is extremely slow and is heavily affected by the speed of the computer. This made most of the demos run at an OK speed, but some of the games may have been too fast.

The new emulator runs somewhere around 100x faster. As a result, the 6502 IDE allows you to set a processor speed used for the emulation.
